# sfx_008 — `error_buzz`

**Archetype:** error / denial / invalid-action
**Shape:** canonical `SfxrParams` per synthesis attempt_001 §3.
**Use cases:** menu item locked, out of mana, action refused, wrong
button in puzzle, invalid-target on cursor
**Variants:** `error_soft`, `error_hard`

## Description

Low square tone with strong vibrato (rate 0.75, depth 0.4) for the
characteristic "buzz" quality. LPF at 0.6 removes ear-piercing highs.
Slight repeatSpeed (0.2) causes a single re-trigger — giving the
perceptual feel of two short "bzzt" pulses.

## Variants

**`error_hard`** (critical failure, alarm-like):
```json
{ "waveType": "square", "envelopeSustain": 0.25, "envelopePunch": 0.5,
  "envelopeDecay": 0.3, "baseFreq": 0.18, "vibratoStrength": 0.55,
  "vibratoSpeed": 0.85, "repeatSpeed": 0.35, "lpFilterResonance": 0.35,
  "masterVolume": 0.55 }
```

Deeper base + stronger vibrato + more retriggers = "klaxon"-ish
alert.
