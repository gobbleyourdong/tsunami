# sfx_014 — `confirm`

**Archetype:** positive menu-confirm / yes-button (primary archetype)
**Shape:** canonical `SfxrParams`.
**Use cases:** dialog-advance, menu-OK, selection-accepted, item-
pickup-confirm, quest-accept
**Variants:** `confirm_soft`, `confirm_major_event`

**Note:** earlier, `menu_confirm` was listed as a variant of sfx_007
`menu_blip`. Promoting to primary here because it's used far more
often than menu_blip proper and warrants distinct tuning.

## Description

Two-note ascending arpeggio (`arpMod: 0.45`, `arpSpeed: 0.75`) — up a
perfect 5th. Square wave, 50% duty, quick decay. Feels like a
cheerful "yep!" — takes ~250ms total. Slight HPF keeps it bright.
