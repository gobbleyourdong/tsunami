# sfx_007 — `menu_blip`

**Archetype:** menu-tick / cursor-move / confirm-light
**Shape:** canonical `SfxrParams` per synthesis attempt_001 §3.
**Use cases:** menu scroll, cursor-position-change, light-tap-
confirmation, page-turn
**Variants:** `menu_confirm` (stronger), `menu_cancel` (descending)

## Description

Very short (~120ms total), high-frequency square-wave tick. 25% duty
for thin character. Slight HPF at 0.1 to strip any low-end rumble.
Low volume (0.3) — menu sounds should be unobtrusive and interrupt-
resistant (can play 4-5 times/sec without annoyance).

## Variants

**`menu_confirm`** (stronger, two-note):
```json
{ "waveType": "square", "envelopeSustain": 0.08, "envelopePunch": 0.3,
  "envelopeDecay": 0.12, "baseFreq": 0.55, "arpMod": 0.4, "arpSpeed": 0.8,
  "duty": 0.5, "masterVolume": 0.4 }
```

Fast arpeggio (2 notes in <100ms) gives "yes-confirm" feel.

**`menu_cancel`** (descending, discouraging):
```json
{ "waveType": "square", "envelopeSustain": 0.08, "envelopePunch": 0.2,
  "envelopeDecay": 0.15, "baseFreq": 0.5, "freqRamp": -0.2,
  "duty": 0.5, "masterVolume": 0.35 }
```

Downward slide. Not painful, just "nope."
